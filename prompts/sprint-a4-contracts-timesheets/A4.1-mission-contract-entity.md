> **DoD** :
> - [ ] Entité `MissionContract` générée auto dès `worker.assignment.accepted`
> - [ ] Champs légaux LSE obligatoires : agence + n° autorisation LSE, client (entreprise utilisatrice), worker, branche CCT, taux, horaires, lieu
> - [ ] Validation pré-création : autorisation LSE valide, permis worker valide, taux ≥ min CCT, LTr OK (pas 50h/sem)
> - [ ] Statut : `draft`, `pending_signatures`, `signed`, `terminated`, `cancelled`

## Fichiers
- `packages/domain/src/contracts/mission-contract.ts`
- `packages/application/src/contracts/generate-mission-contract.use-case.ts`
- Handler webhook `worker.assignment.accepted` → génère contrat

## Validations obligatoires (refus si échec)
- `agency.lseAuthorization.status === 'active'` et `expiresAt > mission.dateTo`
- `worker.workPermit.valid` et `expiresAt > mission.dateTo`
- `worker.employmentContract.active`
- `rate >= cctMinimum(branch, qualification, canton, workerAge, date)`
- `cumulHoursWeek(worker, isoWeek) + missionHours <= 50`
- `restBetweenMissions >= 11h`

## Tests
- Chemin heureux → contrat `draft` créé avec tous les champs légaux.
- Autorisation LSE expirée → refus `LseAuthorizationExpired` + alerte.
- Taux < min CCT → refus `BelowCctMinimum`.
- Dépassement 50h/sem → refus `LtrViolationWeeklyLimit`.
- Isolation tenant.
