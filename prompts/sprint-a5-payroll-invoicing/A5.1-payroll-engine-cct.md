## Structure du moteur
```typescript
// packages/domain/src/payroll/payroll-engine.ts
export class PayrollEngine {
  constructor(
    private readonly cctRates: CctMinimumRateRepository,
    private readonly holidays: CantonHolidaysRepository,
    private readonly clock: Clock,
  ) {}

  computeWeek(input: {
    worker: WorkerSnapshot
    timesheets: Timesheet[]
    isoWeek: WeekIso
    client: ClientSnapshot
    clientContract: ClientContract
  }): Result<PayrollBreakdown, PayrollError>
}

export interface PayrollBreakdown {
  isoWeek: WeekIso
  workerId: StaffId
  lines: PayrollLine[]      // une par jour ou par segment
  grossBaseRappen: bigint
  surchargesRappen: bigint
  grossTotalBeforeCompsRappen: bigint
  computationContext: Json  // snapshot des taux/règles utilisés
}
```

## Cas limites à couvrir (tests)
1. Journée normale 8h, jour ouvré → heures × taux.
2. Journée avec pause 30min non rémunérée correctement décomptée.
3. Heures de nuit (23h-6h) → +25%.
4. Heures dimanche → +50%.
5. Cumul nuit+dimanche (ex. dim 23h-6h lundi) → règles CCT branche.
6. Jour férié cantonal (ex. 1er août) → +100% ou +50% selon branche.
7. Heures sup > contrat hebdo → +25% sauf compensation.
8. Heures > 50h/sem → refus total (erreur domaine, pas de calcul).
9. Semaine avec 2 clients → taux différents, majorations différentes.
10. Changement de barème CCT en milieu de semaine → split par date.
11. Workers < 20 ans → taux différent.
12. Intérimaire qui termine sa mission en milieu de semaine.
13. Timesheet disputé → exclu du calcul.
14. Timesheet validé tacitement → inclus.
15. Mission annulée J-1 → annulation, pas de paie.
16. Pause 0min sur 8h → anomalie déjà remontée en A4, ici le calcul respecte ce qui est saisi.
17. Férié cantonal = férié d'un canton autre que celui de travail → pas applicable.
18. Arrondi 5cts au niveau NET (fait en A5.3), ici on garde précision au centime.
19-30. Combinaisons nuit+sup, dim+sup, dim+férié, ISO week à cheval sur deux mois, etc.
