## Contexte
Un dev doit pouvoir `git clone` + `make up` et tester en 2 minutes. Pas d'installation manuelle locale de Postgres ou Redis.

## Fichiers à créer
- `docker-compose.yml` (racine)
- `Makefile` (racine)
- `apps/mock-moveplanner/` — serveur Express qui expose l'OpenAPI MP en fixtures, signe webhooks HMAC avec secret `dev-mock-secret`
- `.env.example` (racine) — liste exhaustive des variables d'env attendues
- `docs/dev-setup.md` — guide de démarrage

## docker-compose.yml (squelette)
```yaml
services:
  postgres:
    image: postgres:16-alpine
    ports: ["5432:5432"]
    environment: { POSTGRES_USER: dev, POSTGRES_PASSWORD: dev, POSTGRES_DB: interim_dev }
    volumes: [ postgres_data:/var/lib/postgresql/data ]
    healthcheck: { test: ["CMD-SHELL", "pg_isready -U dev"], interval: 5s }
  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]
    healthcheck: { test: ["CMD", "redis-cli", "ping"], interval: 5s }
  mailhog:
    image: mailhog/mailhog
    ports: ["1025:1025", "8025:8025"]
  mock-moveplanner:
    build: ./apps/mock-moveplanner
    ports: ["3030:3030"]
    environment: { MOCK_HMAC_SECRET: dev-mock-secret }
volumes: { postgres_data: }
```

## Mock MovePlanner — endpoints à stubber
- `POST /api/v1/partners/:id/workers` → 200
- `POST /api/v1/partners/:id/workers/:staffId/availability` → 200 `{ accepted, rejected }`
- `POST /api/v1/partners/:id/assignments/:requestId/response` → 200
- `POST /api/v1/partners/:id/timesheets/:id/sign` → 200
- `GET /api/v1/partners/:id/timesheets` → fixtures JSON
- Endpoint admin `POST /_mock/emit-webhook` qui signe et POST vers `API_URL/webhooks/moveplanner`

## Tests
- Script `./scripts/smoke-test.sh` : vérifie que chaque service répond sur son port.
- Test CI-friendly : `make up && curl http://localhost:3000/health` retourne 200.
