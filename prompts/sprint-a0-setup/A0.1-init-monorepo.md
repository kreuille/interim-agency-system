## Contexte

Point de départ du projet. On pose les fondations sur lesquelles tous les autres prompts vont s'appuyer. Architecture hexagonale, TypeScript strict, monorepo pnpm.

Lire en préambule : `CLAUDE.md` (tout), `docs/03-plan-de-dev.md §3 Sprint A.0`, `docs/05-architecture.md §2–§4`.

## Instructions

1. **Init repo + outils** (sur la machine locale) :
   - `git init`, `git branch -m main`, `gh repo create {org}/interim-agency-system --private`
   - Node 20 via `.nvmrc`, pnpm 9 via `corepack enable && corepack prepare pnpm@9 --activate`.

2. **Workspace pnpm** :
   ```yaml
   # pnpm-workspace.yaml
   packages:
     - 'apps/*'
     - 'packages/*'
   ```

3. **tsconfig.base.json** :
   ```json
   {
     "compilerOptions": {
       "target": "ES2022",
       "lib": ["ES2022"],
       "module": "NodeNext",
       "moduleResolution": "NodeNext",
       "strict": true,
       "noImplicitAny": true,
       "strictNullChecks": true,
       "noUncheckedIndexedAccess": true,
       "exactOptionalPropertyTypes": true,
       "noImplicitOverride": true,
       "noFallthroughCasesInSwitch": true,
       "forceConsistentCasingInFileNames": true,
       "skipLibCheck": true,
       "esModuleInterop": true,
       "resolveJsonModule": true,
       "isolatedModules": true,
       "declaration": true,
       "declarationMap": true,
       "sourceMap": true
     }
   }
   ```

4. **ESLint + Prettier** :
   - ESLint flat config (v9+) avec `@typescript-eslint/strict`, `eslint-plugin-import`, `eslint-plugin-security`, `eslint-plugin-unicorn` (sélectif).
   - Règle `no-restricted-imports` : bloquer les imports de `infrastructure` depuis `packages/domain/`.
   - Prettier : printWidth 100, singleQuote, trailingComma all, arrowParens always.

5. **Husky + lint-staged** :
   - Pré-commit : `pnpm lint` sur fichiers stagés + `pnpm typecheck` incremental.

6. **Packages `domain` et `shared`** :
   - `packages/shared/src/money.ts` : class `Money` immuable en Rappen.
   - `packages/shared/src/week-iso.ts` : VO `WeekIso`.
   - `packages/shared/src/result.ts` : type Result<T, E> (ou wrap neverthrow).
   - `packages/shared/src/clock.ts` : interface `Clock` + implémentation `SystemClock`.
   - `packages/domain/src/shared/ids.ts` : `Uuid`, `AgencyId`, `StaffId`.

7. **Apps — squelettes** :
   - `apps/api/src/main.ts` : Express minimal avec `/health` retournant `{ status: 'ok', version: env.VERSION }`.
   - `apps/web-admin` : `npx create-next-app@latest` dans le dossier, App Router, TS, Tailwind.
   - `apps/web-portal` : idem.
   - `apps/worker/src/main.ts` : import BullMQ worker placeholder (pas branché encore).

8. **Docs** :
