## Contexte
Hébergement Suisse obligatoire pour la confiance clients et la simplification nLPD (pas de transferts).

## Livrables
- `docs/adr/0002-hosting-choice.md`
- `infra/terraform/` (si on choisit IaC) — ressources hébergeur + DNS
- `docs/runbooks/infra-provisioning.md`
- `docs/compliance/dpa-{hebergeur}.pdf` (contrat signé)

## Étapes
1. Évaluer Infomaniak Public Cloud vs Exoscale (coûts, SLA, support, Kubernetes vs VM).
2. Signer contrat + DPA.
3. Provisionner : VPC/projet, Postgres managé (Infomaniak "Managed PostgreSQL"), Redis, Object Storage, LB + certs.
4. Configurer DNS (Registrar type Infomaniak/Gandi) pour les 3 sous-domaines.
5. Mettre en place OIDC trust entre GH et l'hébergeur.
6. Créer secret manager et y pousser les secrets initiaux (DB password, HMAC MP, API keys).
7. Test de bout en bout : déploiement d'une image "hello world" depuis GH Actions.
8. Provisionner backup Postgres (snapshot quotidien + WAL archiving).

## Conformité
- DPA avec hébergeur (nLPD art. 9).
- Hébergement **Suisse seulement**, zone CH.
- Chiffrement au repos CMEK activé (géré par hébergeur ou par Cloud KMS).
- Accès infra restreint : MFA obligatoire, logs d'accès conservés.
