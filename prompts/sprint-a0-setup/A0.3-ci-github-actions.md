## ci.yml — jobs
1. **install** : pnpm + cache
2. **lint** : `pnpm lint`
3. **typecheck** : `pnpm typecheck`
4. **test-unit** : `pnpm -r test:unit -- --coverage`, upload Codecov
5. **test-integration** : services Postgres + Redis via `services:` GH, Testcontainers
6. **build** : build toutes les apps, vérifier pas d'erreur
7. **docker-build** : build images `api`, `web-admin`, `web-portal`, `worker` (tag sha)
8. **security-scan** : Trivy sur images + `pnpm audit --audit-level=high`

## release.yml
- Trigger : tag `v*.*.*`
- Steps : build images taguées `vX.Y.Z`, push vers `ghcr.io` ou registry Infomaniak, create GH Release avec changelog auto (via `git-cliff`)

## Branch protection (à configurer côté GitHub)
- `main` : require PR, 1 approving review, status checks required (lint/typecheck/test), dismiss stale reviews, include admins
- `staging` : auto-deploy depuis `main`

## Secrets GitHub à configurer (hors scope du code)
- `REGISTRY_TOKEN`
- `CODECOV_TOKEN`
- `INFOMANIAK_DEPLOY_KEY` (OIDC préférable à long terme)

## Tests
- PR test qui casse volontairement le lint → CI rouge.
- PR test avec test intégration échouant → CI rouge.
- Merge en main déclenche build + tag auto staging.
