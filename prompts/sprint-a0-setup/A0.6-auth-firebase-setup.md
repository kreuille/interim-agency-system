## Contexte
Sans auth robuste, pas de multi-tenant, pas de conformité. À faire tôt pour que tous les autres prompts s'appuient dessus.

## Choix à trancher dans l'ADR
- **Firebase Auth Identity Platform** : aligné avec MovePlanner (si MP utilise GCP). MFA TOTP & SMS. Multi-tenant natif.
- **Supabase Auth** : open source, aligné stack Postgres, MFA TOTP. Moins mature sur le multi-tenant "hard".

Recommandation par défaut : **Firebase Identity Platform** pour l'alignement MP et la maturité.

## Tests
- User sans JWT → 401.
- JWT expiré → 401.
- User agence A → requête à `/workers/{idAgenceB}` → 404 (pas 403, évite leak).
- User `viewer` → POST /workers → 403.
- Fondateur login → enrollement MFA obligatoire → accès après.
- Magic link intérimaire : email reçu, click → session créée, expire 15 min.
