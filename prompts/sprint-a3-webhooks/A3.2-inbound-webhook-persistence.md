## Schéma
```
inbound_webhook_events
  id uuid pk
  external_event_id text unique not null
  event_type text not null
  raw_payload jsonb not null
  signature text not null
  received_at timestamptz default now()
  processed_at timestamptz null
  attempts int default 0
  last_error text null
  status enum(received, processing, processed, error, dead)
```

## Architecture
```
POST /webhooks/moveplanner → verify HMAC → INSERT inbound_webhook_events (ON CONFLICT do nothing)
  → respond 200
  → enqueue BullMQ job { event_id }

Worker mp-webhook-dispatch:
  SELECT ev WHERE id=event_id AND status IN ('received','error')
  UPDATE status='processing'
  dispatch → handler (A3.3+)
  UPDATE status='processed', processed_at=now()
  ou UPDATE status='error', attempts++, last_error
  après 5 échecs → status='dead', alert
```

- Event déjà existant (retry MP) → pas de doublon, 200.
- Handler qui throw → attempts++.
- 5 échecs → DLQ.
