## Points clés
- Le tenant a un **cycle de vie** (`onboarding` → `trial` → `active` → `suspended` → `archived`) avec transitions testées.
- Provisioning transactionnel : création tenant + compte admin + bucket + secrets = rollback total si une étape échoue.
- Un tenant `archived` garde ses données 30j (obligation réversibilité nLPD + SLA), puis purge après export data.

## Tests
- 50+ scénarios cross-tenant (voir aussi B2.2).
- Transaction rollback : simuler échec sur step N → tout rollback.
- Performance : provisioning en < 3s pour ne pas bloquer le user.
