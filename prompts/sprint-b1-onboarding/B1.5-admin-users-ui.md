# B1.5 — Admin Users UI (Paramètres → Utilisateurs)

> **Sprint** : B.1 — **Effort** : M — **BlockedBy** : B1.3
> **Skills** : `skills/dev/frontend-next`

## DoD
- [ ] Page `/settings/users` accessible aux rôles `agency_admin` uniquement
- [ ] Liste des utilisateurs du tenant avec : email, nom, rôle, dernière connexion, statut (active/invited/suspended)
- [ ] Inviter : formulaire email + rôle → email d'invitation avec magic link + création compte dans le tenant
- [ ] Modifier rôle : dropdown avec confirmation (warning si retrait `agency_admin`)
- [ ] Désactiver compte : soft delete, révocation session Firebase, user bloqué au prochain login
- [ ] Réactiver compte désactivé
- [ ] Empêcher désactivation du dernier `agency_admin` (au moins 1 admin actif obligatoire)
- [ ] Audit log à chaque modification (création, invitation, changement rôle, désactivation)
- [ ] Tests E2E Playwright

## UX
- Affichage en table responsive, tri par rôle + dernière connexion.
- Invite → côté serveur : Firebase tenant user created + custom claim `role` posé + magic link envoyé (B1.4).
- Clarté des rôles : tooltip sur chaque rôle avec description courte (extrait `docs/07-rôles.md §2`).

## Tests
- Inviter un user, il reçoit magic link, clique, compte actif avec le bon rôle.
- Désactiver le dernier admin → erreur explicite.
- Changer rôle user actuellement connecté → session invalidée, reconnexion forcée.
- Cross-tenant : tenter invit user d'un autre tenant → 403.
