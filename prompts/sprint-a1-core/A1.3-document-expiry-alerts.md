## Fichiers à créer
- `apps/worker/src/jobs/document-expiry-scan.job.ts`
- `packages/application/src/compliance/document-alert.service.ts`
- Tables : `document_alerts` (documentId, alertType, sentAt, channel)

## Logique
```
tous les docs WHERE status=valid AND expiresAt BETWEEN today AND today+threshold
  → pour chaque : générer alerte si pas déjà envoyée ce jour
  → email gestionnaire + SMS intérimaire + webhook sortant
  → log entry document_alerts
tous les docs WHERE status=valid AND expiresAt < today
  → passage status=expired
  → alerte critique
  → blocage missions à venir de cet intérimaire
```

## Tests
- Doc avec expiresAt = J+30 et type=permit_B → alerte le jour J.
- Doc qui a déjà reçu alerte J-60 → pas d'alerte à J-30 sauf nouveau seuil.
- Doc expiré → status=expired, missions en cours signalées.
- Cross-tenant : l'alerte n'est envoyée qu'au gestionnaire de la bonne agence.
