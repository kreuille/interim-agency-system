## Endpoints
- `POST /api/v1/workers/:id/documents` (multipart) → 202 `{ documentId, scanStatus: pending }`
- `GET /api/v1/workers/:id/documents` → liste paginée
- `GET /api/v1/workers/:id/documents/:docId/download` → URL signée 15min
- `PATCH /api/v1/workers/:id/documents/:docId/validate` → validation manuelle gestionnaire
- `DELETE /api/v1/workers/:id/documents/:docId` → soft delete

## Flow upload
```
1. Client POST multipart
2. Validate MIME magic bytes (pdf, jpeg, png, heic)
3. Upload direct vers Object Storage (via signed URL ou stream serveur)
4. Insert WorkerDocument row (status=pending_scan)
5. Enqueue ClamAV scan job
6. Scan done: status=pending_validation ou rejected
7. Gestionnaire valide depuis UI → status=valid
```

## Tests
- Upload PDF valide → 202 → scan OK → valid après validation.
- Upload EXE déguisé en PDF (magic bytes ≠) → 400.
- Upload > 10 MB → 413.
- Download cross-tenant → 404.
- Document expiré → status=expired au cron.
- Audit log écrit sur chaque étape.
