## Modèle
```typescript
interface AvailabilitySlot {
  dateFrom: Date          // UTC
  dateTo: Date
  status: 'available' | 'tentative' | 'unavailable'
  reason?: string
  source: 'internal' | 'worker_self' | 'api'
  rrule?: string          // RFC 5545
}
interface WorkerAvailability {
  id, workerId, agencyId,
  slots: AvailabilitySlot[],
  freshness: 'realtime' | 'cached' | 'stale',
  ttlExpiresAt: Date,
  lastUpdatedAt: Date
}
```

## API
- POST /api/v1/workers/:id/availability (remplacement partiel d'une période)
- DELETE /api/v1/workers/:id/availability/:slotId
- GET /api/v1/workers/:id/availability?from=&to=
- GET /api/v1/agencies/:id/availability/week/:isoWeek (vue équipe)

## Tests
- Chevauchement dispo+indispo → indispo prime.
- Récurrence hebdo mercredi → instance correctement étendue sur 4 semaines.
- Freshness : slot > TTL → freshness=stale, events émis.
